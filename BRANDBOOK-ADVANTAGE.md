**Signature Element:** 15¬∞ diagonal cuts and transitions

**Usage:**
- Section transitions (diagonal wipes between backgrounds)
- Image treatments (15¬∞ rotated frames)
- Color gradient overlays (diagonal direction)

**Rules:**
- 80% of layout follows strict grid (trust, professionalism)
- 20% breaks grid strategically (energy, differentiation)
- Hero images can bleed outside containers

## üé¨ MOTION & INTERACTION DESIGN

### Animation Principles

#### Core Motion Philosophy: "DELIBERATE MOMENTUM"

**Not:** Bouncy, playful, excessive animations
**Instead:** Purposeful, confident, forward-moving transitions

**Duration Standards:**
- **Micro (100-200ms):** Hover states, button feedback
- **Short (200-400ms):** Element entrances, transitions
- **Medium (400-600ms):** Section reveals, scroll-triggered
- **Long (600-800ms):** Page transitions, modals

**Easing Functions:**
```css
/* Primary: Confident Forward */
--ease-advantage: cubic-bezier(0.4, 0.0, 0.2, 1);

/* Secondary: Smooth Deceleration */
--ease-smooth: cubic-bezier(0.25, 0.1, 0.25, 1);

/* Attention: Bounce-back (sparingly) */
--ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

### Scroll-Triggered Animations

**Section Entrance:**
```
Trigger: Element enters viewport (20% visible)
Animation:
  1. Fade in (0 ‚Üí 1 opacity)
  2. Translate up (32px ‚Üí 0px)
  3. Duration: 600ms
  4. Easing: ease-advantage
  5. Stagger: 100ms delay per child element
```

**Parallax Scrolling (Subtle):**
```
Application: Background elements only
Speed: 0.3x-0.5x of scroll speed (slow subtle movement)
Usage: Accent shapes, gradient overlays
Avoid: Text or critical content (accessibility concern)
```

**Progress Indicators:**
```
Reading Progress Bar:
- Fixed top of viewport
- Height: 4px
- Color: Ignition Orange
- Animation: Smooth width update (no jank)
```

## üìè DESIGN SYSTEM GOVERNANCE

### Usage Rules & Guidelines

#### DO's ‚úÖ

**Color:**
‚úÖ Use Future Navy for trust/credibility moments
‚úÖ Use Ignition Orange for action/energy/urgency
‚úÖ Maintain 4.5:1 minimum contrast ratio for text
‚úÖ Use color combinations from approved pairings list
‚úÖ Apply diagonal gradients at 165¬∞ angle

**Typography:**
‚úÖ Use Inter exclusively (no font mixing)
‚úÖ Maintain 1.5x minimum size difference between hierarchy levels
‚úÖ Use SemiBold (600) or Bold (700) for emphasis
‚úÖ Keep line length between 50-75 characters for readability
‚úÖ Apply 1.6-1.7 line-height for body copy

**Spacing:**
‚úÖ Use 8px increment system strictly
‚úÖ Maintain consistent vertical rhythm (24px paragraph spacing)
‚úÖ Apply generous white space (96px+ between major sections)
‚úÖ Keep content max-width at 1280px for readability

**Components:**
‚úÖ Use pre-defined component variants from library
‚úÖ Maintain component padding/spacing standards
‚úÖ Apply hover states with 300ms transitions
‚úÖ Include accessibility features (focus states, ARIA labels)

#### DON'Ts ‚ùå

**Color:**
‚ùå Never use black (#000000) for text (use Slate 900 instead)
‚ùå Don't create new color variations without system approval
‚ùå Avoid low-contrast combinations (check WCAG compliance)
‚ùå Don't use more than 3 colors per component
‚ùå Never use pure white (#FFFFFF) for backgrounds (use Elevation White)

**Typography:**
‚ùå Never use more than 3 font weights per page
‚ùå Don't use ALL CAPS for body text (only labels/eyebrows)
‚ùå Avoid justified text alignment (use left-aligned)
‚ùå Don't use underlines for emphasis (reserve for links)
‚ùå Never use font sizes smaller than 14px for body content

**Spacing:**
‚ùå Don't use arbitrary spacing values (stick to 8px system)
‚ùå Avoid cramped layouts (<24px between related elements)
‚ùå Don't let content exceed 800px width for body text
‚ùå Never ignore mobile spacing (minimum 20px margins)

**Components:**
‚ùå Don't create one-off component variations
‚ùå Avoid mixing component styles from different sections
‚ùå Don't skip hover/focus states for interactive elements
‚ùå Never sacrifice accessibility for visual aesthetics

## üéì DESIGN RATIONALE: WHY THIS SYSTEM WORKS

### Psychological Foundations

**1. Future Navy + Ignition Orange Palette**
- **Navy:** Triggers trust and credibility (association with corporate stability, Prisma legacy)
- **Orange:** Creates urgency and action (warm, energetic, youthful without being childish)
- **Contrast:** High-energy orange against stable navy creates "tension" that mirrors "future torn" concept
- **Parent Psychology:** Navy calms anxiety, Orange enables pride ("my child is taking action")

**2. Diagonal 15¬∞ Energy System**
- **Neuroscience:** Diagonal lines increase eye movement and attention (vs static horizontal/vertical)
- **Metaphor:** Diagonal suggests upward trajectory, growth, breaking from status quo
- **Visual Differentiation:** Creates unique recognizable brand pattern distinct from competitors
- **Movement:** Suggests forward momentum without chaos (controlled energy)

**3. Inter Typography System**
- **Legibility:** Optimized for digital screens (critical for landing page conversion)
- **Modern Professionalism:** Contemporary without trendy (timeless, credible)
- **Weight Range:** 9 weights allow hierarchy without font-switching (system consistency)
- **Accessibility:** Excellent x-height and letter spacing for Spanish diacritics

**4. "Credible Aspiration" Photography**
- **Authenticity:** Real (not stock) builds trust with skeptical parents
- **Aspiration:** Slightly elevated environments trigger "my child could do this"
- **Relatability:** Diverse representation ensures broad audience sees themselves
- **Evidence:** Close-ups of work prove tangible outcomes (not just smiling faces)

### Conversion Optimization Built-In

**Trust Signals Hierarchy:**
1. **Immediate (0-3 seconds):** Logo + Corporate logos + "By Prisma"
2. **Engagement (3-10 seconds):** Mentor photos + Company names + Real student projects
3. **Consideration (10-60 seconds):** Testimonials with specifics + Metrics (85% completion)
4. **Decision (60+ seconds):** FAQ addressing objections + Scholarship availability

**CTA Strategy:**
- **Ignition Orange:** Creates visual pop (can't be missed)
- **Action Language:** "Aplicar Ahora" vs vague "Learn More"
- **Repetition:** Primary CTA appears 4√ó (above fold, mid-page, pre-FAQ, final push)
- **De-risking:** "Becas disponibles" + "Respuesta en 48hrs" lowers commitment fear

**Mobile-First Considerations:**
- **Touch Targets:** 44px minimum (fat-finger friendly)
- **Thumb Zone:** Primary actions in bottom third (natural reach)
- **Progressive Disclosure:** Forms break into steps (less overwhelming)
- **Speed:** WebP images + lazy loading (Peru mobile data concerns)
